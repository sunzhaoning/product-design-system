# 市场任务管理系统

> **版本**：v1.1  
> **日期**：2026-03-13  
> **状态**：草案

---

## 问题

1. **任务分散难以追踪**：任务信息分散在 Excel 和微信群，进度不透明，市场总监和部门经理无法实时掌握整体进展；
2. **缺乏任务关联**：任务之间没有依赖关系，销售人员不清楚哪些任务需要先完成，容易遗漏关键节点；
3. **进度更新随意**：缺乏审批机制，进度数据可信度低，难以支撑业绩考核；
4. **复杂任务难管理**：大型项目无法拆解，缺少层级结构，责任划分不清晰；
5. **年度数据难统计**：无法按年度周期统计业绩数据，考核缺乏数据支撑。

---

## 需求设计

### 一、年度配置

年度配置用于定义业绩考核的时间周期，支持按年度统计任务完成情况。

1. **年度列表**：
   - 展示所有年度配置，包括年度名称、起止时间、状态；
   - 卡片展示当前活跃年度的关键指标（客户数、任务数、完成率）；
   - 列表支持按状态筛选（全部/活跃/归档）。

2. **年度创建**：
   - 点击「新建年度」按钮，打开创建弹窗；
   - **年度名称** — 文本输入，必填，如"2026年度"；
   - **开始日期** — 日期选择，必填；
   - **结束日期** — 日期选择，必填，需晚于开始日期；
   - **设为活跃** — 开关，开启后自动将其他年度设为非活跃（同时仅一个活跃年度）。

3. **年度激活**：
   - **如果当前年度已激活**，点击「激活」按钮，提示"是否将 {年度名称} 设为活跃年度？原活跃年度将变为正常状态"；
   - **如果确认**，更新年度状态，刷新列表；
   - 同一时间只能有一个活跃年度。

4. **年度归档**：
   - 点击「归档」按钮，确认后年度状态变为"已归档"；
   - 归档年度的数据仅可查看，不可编辑。

5. **权限说明**：
   - 仅系统管理员可创建、编辑、激活、归档年度配置；
   - 其他角色仅可查看年度配置。

6. **操作日志**：
   - 创建年度：记录操作人、年度名称、起止时间；
   - 激活年度：记录操作人、原活跃年度、新活跃年度；
   - 归档年度：记录操作人、归档时间。

### 二、客户管理

客户管理用于维护市场部跟进的客户信息，每个客户下可创建多个任务。

1. **客户列表**：
   - 树形卡片展示，客户为一级节点，任务为二级及以下节点；
   - 支持按客户名称搜索、按状态筛选（全部/正常/暂停/流失）；
   - 卡片展示客户名称、联系人、任务数、最新任务进度；
   - 点击客户卡片，展开显示该客户下的任务树。

2. **客户新增**：
   - 点击「新建客户」按钮，打开创建弹窗；
   - **客户名称** — 文本输入，必填，全局唯一；
   - **客户编码** — 文本输入，选填；
   - **联系人** — 文本输入，选填；
   - **联系电话** — 文本输入，选填；
   - **地址** — 文本输入，选填；
   - **备注** — 多行文本，选填；
   - **如果客户名称重复**，提示"> 客户名称已存在，请修改"。

3. **客户编辑**：
   - 点击客户卡片上的「编辑」按钮，打开编辑弹窗；
   - 修改客户信息后，点击「保存」按钮更新。

4. **客户状态管理**：
   - 客户状态包括：正常、暂停、流失；
   - **如果状态为"暂停"或"流失"**，客户卡片置灰显示，任务列表提示"> 该客户已暂停跟进，任务暂停执行"；
   - 状态变更需部门经理审批。

5. **客户详情**：
   - 点击客户卡片，进入详情页；
   - 顶部展示客户基础信息；
   - 下方展示任务树（支持展开/折叠）；
   - 右侧展示统计卡片：任务总数、完成数、完成率、最新进度。

6. **权限说明**：
   - 客户经理及以上角色可创建客户；
   - 客户负责人可编辑自己负责的客户；
   - 部门经理可管理本部门所有客户；
   - 市场总监可管理全部客户。

7. **操作日志**：
   - 创建客户：记录操作人、客户名称；
   - 编辑客户：记录操作人、变更字段、变更前后值；
   - 状态变更：记录操作人、原状态、新状态、审批人。

### 三、任务管理（核心）

任务管理是本系统的核心功能，支持6级任务层级、任务依赖关系、进度管理和审批流程。

#### （一）任务视图

任务列表支持两种视图切换：

1. **树形列表（默认）**：
   - 按客户分组，缩进展示任务层级；
   - 支持展开/折叠，默认展开第一级；
   - 任务行显示：任务名称、负责人、状态、进度、计划时间；
   - 状态标签用不同颜色区分（待办-灰色、进行中-蓝色、审批中-橙色、已完成-绿色、已取消-红色）。

2. **甘特图**：
   - 点击页面右上角「甘特图」按钮切换视图；
   - 时间轴横轴，任务纵向排列；
   - 任务条显示计划时间和实际进度；
   - 依赖关系用连线表示，FS（完成-开始）用箭头连接，SS（开始-开始）用双向箭头；
   - 支持拖拽调整任务时间（需权限）。

3. **视图共享**：
   - 两种视图共享同一数据源和筛选条件；
   - 筛选条件包括：客户、负责人、状态、时间范围、任务类型；
   - 切换视图时保持筛选条件不变。

#### （二）任务创建

1. **创建入口**：
   - **如果从客户列表创建**，自动关联该客户；
   - **如果从任务树创建子任务**，自动设置父任务和层级。

2. **创建弹窗**：
   - **任务名称** — 文本输入，必填；
   - **关联客户** — 下拉选择，必填（从客户列表创建时自动填充）；
   - **父任务** — 下拉选择，选填（创建子任务时自动填充，支持最多6级）；
   - **负责人** — 人员选择，必填（默认为创建人）；
   - **参与人** — 多选人员，选填；
   - **任务类型** — 下拉选择，必填（客户拜访、方案制作、投标、合同签订、交付、售后支持、里程碑）；
   - **计划开始日期** — 日期选择，选填；
   - **计划结束日期** — 日期选择，选填（需晚于开始日期）；
   - **优先级** — 单选，必填（高/中/低，默认"中"）；
   - **是否里程碑** — 开关，选填（开启后任务名称前显示里程碑图标）；
   - **任务描述** — 多行文本，选填；
   - **前置任务** — 任务选择，选填（支持多选，设置依赖关系）。

3. **层级校验**：
   - **如果父任务已有5级子任务**，提示"> 已达最大层级（6级），无法继续创建子任务"；
   - **如果父任务状态为"已完成"或"已取消"**，提示"> 父任务已完成/已取消，无法创建子任务"。

4. **依赖校验**：
   - **如果设置前置任务**，自动检测是否形成循环依赖（A→B→C→A）；
   - **如果检测到循环依赖**，提示"> 存在循环依赖，请调整前置任务"。

#### （三）任务编辑

1. **编辑权限**：
   - 任务负责人、参与人、上级领导可编辑任务信息；
   - 已完成任务不可编辑。

2. **编辑内容**：
   - 可修改除客户、父任务外的所有字段；
   - 修改计划时间、负责人、依赖关系需部门经理审批。

#### （四）任务删除

1. **删除条件**：
   - **如果有子任务**，提示"> 该任务有子任务，无法删除"；
   - **如果有依赖关系**（作为前置或后置任务），提示"> 该任务存在依赖关系，无法删除"；
   - **如果状态为"已完成"**，提示"> 已完成任务无法删除"；
   - **如果状态为"进行中"或"审批中"**，提示"> 进行中/审批中任务无法删除，请先取消"。

2. **删除确认**：
   - 点击「删除」按钮，弹出确认框"> 确认删除任务「{任务名称}」？删除后无法恢复"；
   - **如果确认**，删除任务及关联数据（人员、依赖、进度记录）。

#### （五）任务详情

1. **详情页结构**：
   - 顶部：任务名称、状态标签、进度条；
   - 左侧：基础信息卡片（客户、负责人、时间、类型、优先级、描述）；
   - 右侧：子任务列表、依赖关系图、审批记录、进度历史。

2. **子任务列表**：
   - 树形展示，支持展开/折叠；
   - 显示子任务名称、负责人、状态、进度；
   - 点击子任务行，跳转到子任务详情。

3. **依赖关系图**：
   - 可视化展示前置任务和后置任务；
   - 前置任务在上，当前任务在中，后置任务在下；
   - 用连线表示依赖类型（FS/SS）。

4. **审批记录**：
   - 时间线展示，包括审批人、审批结果、审批时间、驳回原因；
   - 点击审批记录，查看详细内容。

5. **进度历史**：
   - 折线图展示进度变化趋势；
   - 列表展示每次进度更新的时间、原进度、新进度、说明、审批人。

#### （六）任务层级与进度汇总

1. **层级结构**：
   ```
   客户：XX科技公司
   └── 主任务：年度销售计划（Level 1，摘要任务）
       ├── 售前阶段（Level 2，摘要任务）
       │   ├── 首次拜访（Level 3，叶子任务）
       │   ├── 需求调研（Level 3，叶子任务）
       │   └── 方案编写（Level 3）
       │       ├── 技术方案（Level 4，叶子任务）
       │       └── 商务报价（Level 4，叶子任务）
       ├── 投标阶段（Level 2）
       └── 交付阶段（Level 2）
   ```

2. **摘要任务与叶子任务**：
   - **摘要任务** — 有子任务的父任务，进度由子任务自动汇总，不可手动修改；
   - **叶子任务** — 无子任务的末端任务，进度由负责人手动更新（需审批）。

3. **进度汇总规则**：
   - 父任务进度 = 所有直接子任务进度的平均值；
   - 子任务进度变更后，自动更新父任务进度；
   - 进度汇总向上传递至顶级任务。

4. **状态联动**：
   - **如果所有子任务完成** → 父任务自动标记"已完成"；
   - **如果任意子任务未完成** → 父任务保持"进行中"。

#### （七）任务依赖关系

1. **依赖类型**：
   - **FS（完成-开始）** — 前置任务完成后，后置任务才能开始；
   - **SS（开始-开始）** — 前置任务开始后，后置任务才能开始。

2. **依赖检查**：
   - **开始任务时**，校验前置任务状态：
     - **如果依赖类型为FS**，前置任务需"已完成"；
     - **如果依赖类型为SS**，前置任务需"进行中"或"已完成"；
   - **如果前置任务未满足**，提示"> 前置任务「{任务名称}」未完成，无法开始"。

3. **滞后时间**：
   - 支持设置依赖滞后天数（如：A完成3天后B开始）；
   - **如果有滞后时间**，后置任务的最早开始时间 = 前置任务完成时间 + 滞后天数。

4. **依赖提醒**：
   - **前置任务完成/开始时**，自动通知后置任务负责人；
   - 通知内容："> 前置任务「{任务名称}」已完成，您的任务「{任务名称}」可以开始了"。

#### （八）权限说明

1. **数据权限**：
   - 市场总监：全部任务；
   - 部门经理：本部门任务；
   - 客户经理：自己负责的客户下的任务 + 自己参与的任务；
   - 销售人员：自己负责或参与的任务。

2. **操作权限**：
   - 创建任务：客户经理及以上；
   - 编辑任务：任务负责人、参与人、上级领导；
   - 删除任务：任务负责人（需满足删除条件）；
   - 开始任务：任务负责人；
   - 更新进度：任务负责人；
   - 申请完成：任务负责人；
   - 取消任务：任务负责人（待办状态）、上级领导（进行中/审批中状态）。

#### （九）操作日志

1. **创建任务**：记录操作人、任务名称、关联客户、负责人；
2. **编辑任务**：记录操作人、变更字段、变更前后值；
3. **删除任务**：记录操作人、删除时间、任务名称；
4. **开始任务**：记录操作人、开始时间；
5. **更新进度**：记录操作人、原进度、新进度、进度说明、审批人；
6. **完成任务**：记录操作人、完成时间、审批人；
7. **取消任务**：记录操作人、取消原因；
8. **设置依赖**：记录操作人、前置任务、后置任务、依赖类型。

### 四、进度管理与审批流程

#### （一）进度更新流程

1. **提交进度**：
   - 任务负责人在任务详情页点击「更新进度」按钮；
   - 弹窗显示当前进度，填写新进度（滑块，0-100%，只能增加）；
   - **进度说明** — 多行文本，必填，描述工作内容和成果；
   - 点击「提交」按钮，发起审批申请。

2. **进度限制**：
   - **只能增加，不可回退**（除驳回场景）；
   - **如果新进度 < 当前进度**，提示"> 进度只能增加，无法减少"。

3. **自动完成**：
   - **如果进度更新到100%且审批通过**，自动触发"任务完成"审批流程。

#### （二）审批流程

1. **审批类型**：
   - 进度更新审批；
   - 任务完成审批；
   - 任务取消审批。

2. **审批人配置**：
   - 默认审批人：直属上级；
   - 可配置多级审批（1-3级）；
   - 可指定审批人类型：直属上级、指定人员、指定角色。

3. **审批处理**：
   - 审批人在待办列表或消息通知中点击「去审批」；
   - 查看进度说明、历史记录；
   - **通过** — 点击「通过」按钮，进度生效，自动汇总到父任务；
   - **驳回** — 点击「驳回」按钮，填写驳回原因，退回提交人修改。

4. **驳回处理**：
   - 任意一级驳回 → 退回提交人；
   - 提交人修改后重新提交 → 重新走完整审批流程（从第一级开始）。

5. **审批超时**：
   - **如果审批超过3天未处理**，自动提醒审批人；
   - **如果审批超过7天未处理**，自动升级到上级领导。

#### （三）权限说明

1. **提交权限**：任务负责人；
2. **审批权限**：按配置的审批流程，默认为直属上级；
3. **查看权限**：任务负责人、参与人、上级领导。

#### （四）操作日志

1. **提交审批**：记录提交人、审批类型、提交时间；
2. **审批通过**：记录审批人、审批时间、审批意见；
3. **审批驳回**：记录审批人、驳回时间、驳回原因；
4. **重新提交**：记录提交人、重新提交时间。

### 五、统计报表

#### （一）年度概览

1. **数据卡片**：
   - 客户数、任务数、完成率、平均耗时；
   - 与去年同期对比（增长率）。

2. **趋势图**：
   - 按月展示任务创建数、完成任务数；
   - 支持切换年度查看。

3. **人员绩效排行**：
   - 按完成任务数、完成率排序；
   - 点击人员行，查看该人员的任务明细。

#### （二）客户分析

1. **客户列表**：
   - 展示客户名称、任务数、完成率、最新进度；
   - 支持按完成率排序。

2. **任务类型分布**：
   - 饼图展示各类型任务占比；
   - 点击扇形，查看该类型的任务列表。

#### （三）权限说明

1. 市场总监：查看全部数据；
2. 部门经理：查看本部门数据；
3. 客户经理/销售人员：查看自己参与的任务数据。

### 六、消息通知

1. **通知场景**：
   - 任务分配：通知负责人"> 您有新任务「{任务名称}」，请及时处理"；
   - 依赖满足：前置任务完成/开始时，通知后置任务负责人；
   - 待审批：通知审批人"> 您有 {N} 条待审批任务"；
   - 审批结果：通知提交人"> 您的任务「{任务名称}」进度更新已通过/驳回"；
   - 审批超时：提醒审批人和上级领导。

2. **通知方式**：
   - 系统内消息（必选）；
   - 邮件通知（可选，用户自行配置）；
   - 企业微信/钉钉（可选，需对接）。

### 七、异常场景处理

1. **前置任务被取消**：
   - **如果前置任务被取消**，后置任务的依赖关系自动解除；
   - 通知后置任务负责人"> 前置任务「{任务名称}」已取消，依赖关系已解除"。

2. **负责人离职/调岗**：
   - **如果任务负责人离职**，任务自动转移给上级领导；
   - 上级领导需重新分配负责人。

3. **客户流失**：
   - **如果客户状态变更为"流失"**，该客户下所有进行中任务自动暂停；
   - 任务状态变为"已暂停"，不可操作，需上级领导重新激活。

4. **年度归档**：
   - **如果年度归档**，该年度下的任务数据仅可查看，不可编辑；
   - 未完成任务自动标记"已关闭"。

## 术语表

| 术语 | 说明 |
|------|------|
| 摘要任务 | 有子任务的父任务，进度由子任务自动汇总 |
| 叶子任务 | 无子任务的末端任务，进度手动更新 |
| 前置任务 | 依赖关系中必须先完成/开始的任务 |
| 后置任务 | 依赖关系中依赖其他任务的任务 |
| FS | Finish-to-Start，完成-开始依赖 |
| SS | Start-to-Start，开始-开始依赖 |
| 活跃年度 | 当前正在使用的年度周期 |
