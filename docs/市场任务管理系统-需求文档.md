# 市场任务管理系统

> **版本**：v2.0  
> **日期**：2026-03-14  
> **状态**：正式版  
> **作者**：PM Agent

---

## 问题

1. **任务分散难以追踪**：任务信息分散在 Excel 和微信群，进度不透明，市场总监和部门经理无法实时掌握整体进展；
2. **缺乏任务关联**：任务之间没有依赖关系，销售人员不清楚哪些任务需要先完成，容易遗漏关键节点；
3. **进度更新随意**：缺乏审批机制，进度数据可信度低，难以支撑业绩考核；
4. **复杂任务难管理**：大型项目无法拆解，缺少层级结构，责任划分不清晰；
5. **年度数据难统计**：无法按年度周期统计业绩数据，考核缺乏数据支撑。

---

## 需求设计

### 核心数据结构

系统采用**三层结构**管理市场任务：

```
客户（全局唯一实体）
  ↓ 1:N
年度任务计划（客户×年度的组合容器）
  ↓ 1:N
任务WBS（6级层级树）
```

**关键规则**：
1. **客户**：独立实体，全局唯一（如"XX科技公司"只有一条记录）
2. **年度**：全局唯一的统计维度，可配置起止时间
3. **年度任务计划**：客户×年度的组合，每个客户每年有一个独立的任务计划容器
4. **任务层级**：每个年度任务计划下独立成树，Level 1-6
5. **任务依赖**：仅支持同一客户下的任务建立依赖（可跨年度，不可跨客户）

**数据结构示例**：
```
客户：XX科技公司（全局唯一）
  ├─ 2026年度任务计划（容器）
  │  └─ 2026年度销售计划（L1）
  │     ├─ 售前阶段（L2）
  │     │  ├─ 首次拜访（L3）
  │     │  ├─ 需求调研（L3）
  │     │  └─ 方案编写（L3）
  │     ├─ 投标阶段（L2）
  │     └─ 交付阶段（L2）
  │
  ├─ 2025年度任务计划（容器）
  │  └─ 2025年度维护项目（L1）
  │     └─ 定期回访（L2）
  │
  └─ 2024年度任务计划（容器）
     └─ ...
```

### 一、年度配置

年度配置用于定义业绩考核的时间周期，支持按年度统计任务完成情况。

#### 1. 年度列表

- 展示所有年度配置，包括年度名称、起止时间、状态；
- 卡片展示当前活跃年度的关键指标（客户数、任务数、完成率）；
- 列表支持按状态筛选（全部/活跃/归档）。

#### 2. 年度创建

- 点击「新建年度」按钮，打开创建弹窗；
- **年度名称** — 文本输入，必填，如"2026年度"；
- **开始日期** — 日期选择，必填；
- **结束日期** — 日期选择，必填，需晚于开始日期；
- **设为活跃** — 开关，开启后自动将其他年度设为非活跃（同时仅一个活跃年度）。

#### 3. 年度激活

- **如果当前年度已激活**，点击「激活」按钮，提示"> 是否将 {年度名称} 设为活跃年度？原活跃年度将变为正常状态"；
- **如果确认**，更新年度状态，刷新列表；
- 同一时间只能有一个活跃年度。

#### 4. 年度归档

- 点击「归档」按钮，确认后年度状态变为"已归档"；
- 归档年度的数据仅可查看，不可编辑；
- **归档前检查**：
  - 如果年度下有未完成的任务，提示"> 该年度下有 {N} 个未完成任务，归档后将自动关闭，是否继续？"
  - 如果确认，所有未完成任务自动标记为"已关闭"，进度冻结；
- **归档后影响**：
  - 所有任务变为只读状态，不可编辑、删除、更新进度；
  - 不可以创建新的任务计划；
  - 统计报表仍可查看，数据冻结。

#### 5. 年度归档提醒（P1）

- 系统自动检测已结束但未归档的年度；
- **如果年度结束日期超过30天且未归档**，自动发送提醒给系统管理员；
- 提醒内容："> 年度「{年度名称}」已结束超过30天，建议及时归档"；
- 提醒频率：每周一次，直到归档为止。

#### 6. 权限说明

- 仅系统管理员可创建、编辑、激活、归档年度配置；
- 其他角色仅可查看年度配置。

#### 7. 操作日志

- 创建年度：记录操作人、年度名称、起止时间；
- 激活年度：记录操作人、原活跃年度、新活跃年度；
- 归档年度：记录操作人、归档时间。

### 二、客户管理

客户是独立的业务实体，全局唯一。每个客户可以有多个年度任务计划，每个年度任务计划下有独立的任务WBS树。

#### 1. 客户列表

- 卡片或列表形式展示所有客户；
- 支持按客户名称搜索、按状态筛选（全部/正常/暂停/流失）；
- 卡片展示：客户名称、客户负责人、任务数（进行中/已完成）、最新进度；
- **如果客户状态为"暂停"或"流失"**，卡片置灰显示；
- 快捷操作：「查看详情」「编辑」「新建任务」。

#### 2. 客户创建

- 点击页面右上角「新建客户」按钮，打开创建弹窗；
- **客户名称** — 文本输入，必填，全局唯一；
- **客户编码** — 文本输入，选填；
- **客户负责人** — 人员选择，必填（默认为创建人）；
- **联系人** — 文本输入，选填；
- **联系电话** — 文本输入，选填；
- **地址** — 文本输入，选填；
- **备注** — 多行文本，选填；
- **自动创建年度任务计划** — 开关，默认勾选（自动为当前活跃年度创建任务计划容器）；
- **如果客户名称重复**，提示"> 客户名称已存在，请修改"。

#### 3. 客户编辑

- 点击客户卡片上的「编辑」按钮，打开编辑弹窗；
- 修改客户信息后，点击「保存」按钮更新；
- **客户负责人变更**：
  - 可选择"是否同时变更该客户下所有任务的负责人"；
  - 如果选择"是"，该客户下所有任务的负责人自动变更为新负责人；
  - 变更后通知所有相关人员。

#### 4. 客户状态管理

- 客户状态包括：正常、暂停、流失；
- **如果状态变更为"暂停"**：
  - 该客户下所有进行中任务自动标记为"已暂停"；
  - 已暂停任务不可操作，需上级领导重新激活；
  - 通知所有任务负责人"> 客户「{客户名称}」已暂停，相关任务已暂停执行"；
- **如果状态变更为"流失"**：
  - 该客户下所有任务（除已完成）自动标记为"已关闭"；
  - 通知所有任务负责人"> 客户「{客户名称}」已流失，相关任务已关闭"；
- 状态变更需部门经理审批。

#### 5. 客户删除

- **如果有任务数据**，提示"> 该客户有任务数据，无法删除。建议使用归档功能"；
- **如果无任务数据**，允许删除；
- 删除确认："> 确认删除客户「{客户名称}」？删除后无法恢复"。

#### 6. 客户详情页

客户详情页是查看和管理客户任务的核心页面。

**页面结构**：
```
┌─────────────────────────────────────────┐
│ 客户基础信息卡片                         │
│ XX科技公司 | 负责人：张三 | 正常         │
├─────────────────────────────────────────┤
│ 年度选择器：[2026年度 ▼] [2025] [2024] │
├─────────────────────────────────────────┤
│ 当前选中年度的任务WBS树                 │
│                                          │
│ ▼ 2026年度销售计划 (L1)                 │
│   ├─ 售前阶段 (L2)                      │
│   │  ├─ 首次拜访 (L3)                   │
│   │  └─ 需求调研 (L3)                   │
│   └─ 投标阶段 (L2)                      │
└─────────────────────────────────────────┘
```

**年度选择器**：
- 顶部横向排列所有年度标签；
- 当前选中的年度高亮显示；
- 点击年度标签，切换显示该年度的任务WBS树；
- **如果某个年度无任务计划**，标签显示为灰色，点击后提示"> 该客户在{年度名称}无任务计划"；
- **懒加载**（P2）：切换年度时异步加载任务数据，缓存已访问年度的数据。

**任务WBS树**：
- 树形展示当前年度的任务层级；
- 支持展开/折叠，默认展开第一级；
- 任务节点显示：任务名称、负责人、状态、进度、计划时间；
- 悬停显示快捷操作：「新建子任务」「编辑」「删除」「查看详情」。

#### 7. 权限说明

- **数据权限**：
  - 系统管理员/市场总监：查看全部客户；
  - 部门经理：查看本部门客户（包含下级部门）；
  - 客户经理：查看自己负责的客户 + 参与任务的客户；
  - 销售人员：查看自己参与任务的客户；
- **操作权限**：
  - 创建客户：客户经理及以上；
  - 编辑客户：客户负责人、上级领导；
  - 状态变更：部门经理及以上（需审批）；
  - 删除客户：仅当客户下无任务时可删除。

#### 8. 操作日志

- 创建客户：记录操作人、客户名称、负责人；
- 编辑客户：记录操作人、变更字段、变更前后值；
- 状态变更：记录操作人、原状态、新状态、审批人、变更原因；
- 删除客户：记录操作人、删除时间、客户名称。

### 三、年度任务计划

年度任务计划是客户×年度的组合容器，每个客户每年有一个独立的任务计划容器。

#### 1. 年度任务计划创建

**自动创建**：
- 新建客户时，默认自动为当前活跃年度创建任务计划；
- 可在创建客户时取消勾选"自动创建年度任务计划"。

**手动创建**：
- 在客户详情页，点击"为其他年度创建任务计划"；
- 选择年度，创建该年度的任务计划容器。

**批量创建**（P2）：
- 在年度概览页，点击"批量创建任务计划"；
- 选择年度，勾选多个客户，批量为这些客户创建该年度的任务计划。

#### 2. 年度任务计划删除

- **如果有任务数据**，提示"> 该年度任务计划有任务数据，无法删除"；
- **如果无任务数据**，允许删除。

#### 3. 权限说明

- 创建：市场总监、部门经理、客户经理（负责的客户）；
- 删除：仅当无任务数据时可删除。

### 四、任务管理（核心）

任务管理是本系统的核心功能，采用**年度任务计划→任务WBS**的两层结构。

#### 1. 任务层级结构

**层级定义**：
- **年度任务计划层**：客户×年度的组合容器；
- **任务层**：每个年度任务计划下独立成树，从Level 1开始；
- **最大深度**：单个任务树最多6级（Level 1 - Level 6）。

**任务类型**：
- **摘要任务**：有子任务的父任务，进度由子任务自动汇总，不可手动修改；
- **叶子任务**：无子任务的末端任务，进度由负责人手动更新（需审批）。

#### 2. 任务视图

**树形列表（默认）**：
- 按年度任务计划分组，每个年度任务计划下缩进展示任务层级；
- 支持展开/折叠，默认展开第一级；
- 任务行显示：任务名称、负责人、状态、进度、计划时间；
- 状态标签用不同颜色区分（待办-灰色、进行中-蓝色、审批中-橙色、已完成-绿色、已取消-红色、已暂停-黄色、已关闭-深灰色）。

**甘特图**：
- 点击页面右上角「甘特图」按钮切换视图；
- 时间轴横轴，任务纵向排列；
- 任务条显示计划时间和实际进度；
- 依赖关系用连线表示；
- 支持拖拽调整任务时间（需权限）。

#### 3. 任务创建

**创建入口**：
- **如果从客户详情页创建**，自动关联该客户和当前选中的年度任务计划；
- **如果从任务树创建子任务**，自动设置父任务和层级。

**创建弹窗**：
- **任务名称** — 文本输入，必填；
- **关联客户** — 下拉选择，必填（从客户详情页创建时自动填充）；
- **年度任务计划** — 下拉选择，必填（从客户详情页创建时自动填充当前年度）；
- **父任务** — 下拉选择，选填（创建子任务时自动填充，支持最多6级）；
- **负责人** — 人员选择，必填（默认为创建人）；
- **参与人** — 多选人员，选填；
- **任务类型** — 下拉选择，必填（客户拜访、方案制作、投标、合同签订、交付、售后支持、里程碑）；
- **计划开始日期** — 日期选择，选填；
- **计划结束日期** — 日期选择，选填（需晚于开始日期）；
- **优先级** — 单选，必填（高/中/低，默认"中"）；
- **是否里程碑** — 开关，选填（开启后任务名称前显示里程碑图标⭐）；
- **任务描述** — 多行文本，选填；
- **前置任务** — 任务选择，选填（支持多选，仅显示同一客户的任务）。

**层级校验**：
- **如果父任务已有5级子任务**，提示"> 已达最大层级（6级），无法继续创建子任务"；
- **如果父任务状态为"已完成"或"已取消"**，提示"> 父任务已完成/已取消，无法创建子任务"。

**时间校验**：
- **如果任务时间超出年度范围（±30天）**，显示警告但不阻止：
  "> 提示：任务计划时间超出年度范围（{年度起止时间}），是否继续？"

**依赖校验**：
- **如果设置前置任务**，自动检测是否形成循环依赖（A→B→C→A）；
- **如果检测到循环依赖**，提示"> 存在循环依赖，请调整前置任务"；
- **如果前置任务属于其他客户**，提示"> 前置任务必须与当前任务属于同一客户"。

#### 4. 任务编辑

- 任务负责人、参与人、上级领导可编辑任务信息；
- 已完成任务不可编辑；
- 可修改除客户、年度任务计划、父任务外的所有字段；
- 修改计划时间、负责人、依赖关系需部门经理审批。

#### 5. 任务删除

**删除条件**：
- **如果有子任务**，提示"> 该任务有子任务，无法删除"；
- **如果有依赖关系**，提示"> 该任务存在依赖关系，无法删除"；
- **如果状态为"已完成"**，提示"> 已完成任务无法删除"；
- **如果状态为"进行中"或"审批中"**，提示"> 进行中/审批中任务无法删除，请先取消"。

**删除确认**：
- 点击「删除」按钮，弹出确认框"> 确认删除任务「{任务名称}」？删除后无法恢复"；
- **如果确认**，删除任务及关联数据（人员、依赖、进度记录）。

#### 6. 任务开始

- **开始前检查**：
  - **如果依赖类型为FS**，前置任务需"已完成"；
  - **如果依赖类型为SS**，前置任务需"进行中"或"已完成"；
  - **如果前置任务未满足**，提示"> 前置任务「{任务名称}」未{完成/开始}，无法开始当前任务"；
- **如果检查通过**，任务状态变为"进行中"，记录开始时间。

#### 7. 任务进度更新

**提交进度**：
- 任务负责人在任务详情页点击「更新进度」按钮；
- 弹窗显示当前进度，填写新进度（滑块，0-100%，只能增加）；
- **进度说明** — 多行文本，必填，描述工作内容和成果；
- 点击「提交」按钮，发起审批申请。

**进度限制**：
- **只能增加，不可回退**（除驳回场景）；
- **如果新进度 < 当前进度**，提示"> 进度只能增加，无法减少"。

**自动完成**：
- **如果进度更新到100%且审批通过**，自动触发"任务完成"审批流程。

#### 8. 任务完成

- 任务进度达到100%后，自动发起"任务完成"审批；
- 审批通过后，任务状态变为"已完成"，记录完成时间；
- **如果所有子任务完成** → 父任务自动标记"已完成"。

#### 9. 任务取消

- **如果任务状态为"待办"**，任务负责人可直接取消；
- **如果任务状态为"进行中"或"审批中"**，需上级领导审批；
- 取消原因必填；
- **如果任务被取消**：
  - 后置任务的依赖关系自动解除；
  - 通知后置任务负责人"> 前置任务「{任务名称}」已取消，依赖关系已解除"。

#### 10. 任务依赖关系

**依赖类型**：
- **FS（Finish-to-Start，完成-开始）** — 前置任务完成后，后置任务才能开始；
- **SS（Start-to-Start，开始-开始）** — 前置任务开始后，后置任务才能开始。

**依赖范围**：
- **仅同一客户**：前置任务和后置任务必须属于同一客户；
- **可跨年度**：同一客户下不同年度的任务可以建立依赖。

**滞后时间**：
- 支持设置依赖滞后天数（如：A完成3天后B开始）；
- **如果有滞后时间**，后置任务的最早开始时间 = 前置任务完成时间 + 滞后天数。

**依赖提醒**：
- **前置任务完成/开始时**，自动通知后置任务负责人；
- 通知内容："> 前置任务「{任务名称}」已完成，您的任务「{任务名称}」可以开始了"。

#### 11. 任务详情页

**页面结构**：
```
┌─────────────────────────────────────────┐
```

### 五、进度管理与审批流程

#### 1. 审批类型

- 进度更新审批；
- 任务完成审批；
- 任务取消审批（进行中/审批中状态）。

#### 2. 审批流程

**提交审批**：
- 任务负责人提交进度更新或完成申请；
- 系统自动根据配置找到审批人（默认为直属上级）。

**审批人配置**：
- 默认审批人：直属上级；
- 可配置多级审批（1-3级）；
- 可指定审批人类型：直属上级、指定人员、指定角色。

**审批处理**：
- 审批人在待办列表或消息通知中点击「去审批」；
- 查看进度说明、历史记录；
- **通过** — 点击「通过」按钮，进度生效，自动汇总到父任务；
- **驳回** — 点击「驳回」按钮，填写驳回原因，退回提交人修改。

**驳回处理**：
- 任意一级驳回 → 退回提交人；
- 提交人修改后重新提交 → 重新走完整审批流程（从第一级开始）。

#### 3. 审批超时

- **如果审批超过3天未处理**，自动提醒审批人；
- **如果审批超过7天未处理**，自动升级到上级领导；
- 提醒方式：系统内消息 + 邮件（可选）。

#### 4. 权限说明

- **提交权限**：任务负责人；
- **审批权限**：按配置的审批流程，默认为直属上级；
- **查看权限**：任务负责人、参与人、上级领导。

#### 5. 操作日志

- 提交审批：记录提交人、审批类型、提交时间；
- 审批通过：记录审批人、审批时间、审批意见；
- 审批驳回：记录审批人、驳回时间、驳回原因；
- 重新提交：记录提交人、重新提交时间。

### 六、任务状态机

任务状态包括：待办、进行中、审批中、已完成、已取消、已暂停、已关闭。

#### 1. 状态流转图

```
待办 → 进行中 → 审批中 → 已完成
  ↓       ↓         ↓
已取消  已取消    已取消
  ↓       ↓         ↓
  └───────┴─────────┘
```

**状态说明**：
- **待办**：任务已创建，尚未开始；
- **进行中**：任务已开始执行；
- **审批中**：进度更新或完成申请已提交，等待审批；
- **已完成**：任务已完成（进度100%且审批通过）；
- **已取消**：任务已取消；
- **已暂停**：客户状态变更为"暂停"时，任务自动暂停；
- **已关闭**：客户状态变更为"流失"或年度归档时，任务自动关闭。

#### 2. 状态转换规则

| 当前状态 | 可转换状态 | 触发条件 |
|---------|-----------|---------|
| 待办 | 进行中 | 点击「开始任务」，前置依赖满足 |
| 待办 | 已取消 | 点击「取消任务」 |
| 进行中 | 审批中 | 提交进度更新或完成申请 |
| 进行中 | 已取消 | 上级领导审批通过 |
| 审批中 | 进行中 | 审批驳回 |
| 审批中 | 已完成 | 审批通过（进度100%） |
| 审批中 | 进行中 | 审批通过（进度<100%） |
| 审批中 | 已取消 | 上级领导审批通过 |
| 已暂停 | 进行中 | 上级领导重新激活 |
| 已关闭 | — | 不可转换 |

### 九、RBAC权限管理

RBAC（基于角色的访问控制）采用**细粒度权限配置**，支持自定义角色、多角色、部门层级、权限继承。

#### 1. 核心概念

```
用户
  ↓ N:N
角色
  ↓ N:N
权限（功能权限 + 数据权限）
  ↓ 1:N
资源（年度、客户、任务、审批等）
```

**权限三维模型**：
- **资源维度**：年度配置、客户、年度任务计划、任务、审批、统计报表、数据导入导出、系统配置、部门管理、角色管理（10种）；
- **操作维度**：创建、查看详情、查看列表、编辑、删除、审批、导出、导入、归档、激活（10种）；
- **数据范围维度**：全部、本部门（包含下级部门）、自定义部门、负责的、参与的、仅自己（6种）。

#### 2. 角色管理

**预设角色（5个）**：
- **系统管理员（ADMIN）**：系统配置、年度配置、权限管理；
- **市场总监（DIRECTOR）**：全局查看、审批、统计报表；
- **部门经理（MANAGER）**：本部门管理、审批、统计；
- **客户经理（CUSTOMER_MANAGER）**：客户管理、任务分配；
- **销售人员（SALES）**：任务执行、进度更新。

**自定义角色**：
- 系统管理员可创建自定义角色；
- 可设置上级角色（自动继承上级角色的所有权限）；
- 可追加或覆盖权限；
- 预设角色不可删除，但可以修改权限。

**角色字段**：
- 角色名称、角色编码（唯一）、上级角色ID（可选）、是否预设角色、状态（启用/停用）。

#### 3. 部门管理

**部门层级**：
- 支持两级部门结构：一级部门 → 二级部门；
- **本部门**数据权限包含下级部门的数据。

**部门字段**：
- 部门名称、上级部门ID（NULL表示一级部门）、部门负责人ID、状态。

**示例**：
```
市场部（一级部门）
├─ 华东区（二级部门）
├─ 华南区（二级部门）
└─ 华北区（二级部门）
```

#### 4. 用户角色分配

**多角色支持**：
- 一个用户可以有多个角色；
- 权限取并集（拥有所有角色的权限）；
- 数据范围取并集（范围更大的优先）。

**有效权限预览**：
- 编辑用户时，显示合并后的有效权限；
- 标注哪些权限是继承的，哪些是自定义的。

#### 5. 权限继承规则

**自动继承**：
- 上级角色自动拥有下级角色的所有权限；
- 子角色可以追加或覆盖权限。

**权限合并算法**：
```
获取角色权限：
1. 获取角色自己的权限
2. 如果有上级角色，递归获取上级角色的权限
3. 合并权限（自己的权限优先级更高）

合并规则：
- 相同资源×操作：数据范围取范围更大的
- 不同资源×操作：取并集
```

#### 6. 细粒度权限配置

**权限配置界面**：
```
┌────────────────────────────────────────┐
│ 角色管理 - 编辑权限                     │
├────────────────────────────────────────┤
│ 【客户】                                │
│ ☑ 查看列表  数据范围：[本部门 ▼]       │
│ ☑ 查看详情  数据范围：[本部门 ▼]       │
│ ☑ 创建      数据范围：[—]              │
│ ☑ 编辑      数据范围：[本部门 ▼]       │
│ ☑ 删除      数据范围：[负责的 ▼]       │
│ ☑ 导出      数据范围：[本部门 ▼]       │
│                                         │
│ 【任务】                                │
│ ☑ 查看列表  数据范围：[本部门 ▼]       │
│ ☑ 创建      数据范围：[—]              │
│ ☑ 编辑      数据范围：[本部门 ▼]       │
│ ☑ 删除      数据范围：[本部门 ▼]       │
│ ☑ 开始      数据范围：[负责的 ▼]       │
│ ☑ 更新进度  数据范围：[负责的 ▼]       │
│ ...                                     │
└────────────────────────────────────────┘
```

**数据范围选择器**：
- 全部、本部门、自定义部门（可选择多个部门）、负责的、参与的、仅自己、无权限。

#### 7. 权限校验流程

```
用户访问资源
  ↓
1. 验证用户身份（登录态）
  ↓
2. 获取用户所有角色
  ↓
3. 合并角色权限（取并集）
  ↓
4. 检查功能权限（是否有该操作权限）
  ↓
5. 检查数据权限（是否在数据范围内）
  ↓
允许访问 / 拒绝访问
```

#### 8. 权限审计日志

- 记录所有权限变更操作；
- 包括：创建角色、修改权限、分配角色、创建部门等；
- 记录操作人、操作时间、变更前后数据。

#### 9. 权限矩阵

**年度配置权限**：
- 系统管理员：查看、创建、编辑、激活、归档；
- 其他角色：仅查看。

**客户权限**：
- 系统管理员/市场总监：全部操作，全部数据；
- 部门经理：全部操作，本部门数据；
- 客户经理：创建、查看、编辑（负责的），本部门数据；
- 销售人员：查看、编辑（参与的，部分字段），参与的数据。

**任务权限**：
- 系统管理员：查看、编辑、删除、取消（全部数据）；
- 市场总监：全部操作，全部数据；
- 部门经理：全部操作，本部门数据；
- 客户经理：创建、查看、编辑、开始、更新进度、申请完成、取消（待办），本部门数据；
- 销售人员：查看、编辑（负责的，部分字段）、开始、更新进度、申请完成，参与的数据。

**审批权限**：
- 系统管理员：查看历史（全部）；
- 市场总监：查看待审、审批、查看历史（全部）；
- 部门经理：查看待审、审批、查看历史（本部门）；
- 客户经理：查看待审、审批、查看历史（本部门）；
- 销售人员：查看历史（自己的）。

#### 10. 权限管理功能（P0）

**角色管理页面**：
- 角色列表（预设 + 自定义）；
- 新建角色；
- 编辑角色权限（细粒度配置）；

### 十一、任务评论（P1）

#### 1. 评论功能

- 支持文字评论；
- 支持@提醒相关人员；

#### 2. 评论通知

### 十六、异常场景处理

#### 1. 前置任务被取消

- 后置任务的依赖关系自动解除；
- 通知后置任务负责人"> 前置任务「{任务名称}」已取消，依赖关系已解除"。

#### 2. 负责人离职/调岗

- **如果任务负责人离职**：
  - 任务自动转移给上级领导或客户负责人；
  - 上级领导需在3个工作日内重新分配负责人；
  - 通知相关人员"> 任务「{任务名称}」负责人已变更，请及时处理"。

#### 3. 客户状态变更

- **如果客户状态变更为"暂停"**：
  - 该客户下所有进行中任务自动标记为"已暂停"；
  - 已暂停任务不可操作，需上级领导重新激活；
  - 通知所有任务负责人"> 客户「{客户名称}」已暂停，相关任务已暂停执行"；
- **如果客户状态变更为"流失"**：
  - 该客户下所有任务（除已完成）自动标记为"已关闭"；
  - 通知所有任务负责人"> 客户「{客户名称}」已流失，相关任务已关闭"。

#### 4. 年度归档

- **如果年度归档**：
  - 该年度下的任务数据仅可查看，不可编辑；
  - 未完成任务自动标记"已关闭"，进度冻结。

#### 5. 跨任务树依赖断裂

- **如果前置任务所属的任务树被整体取消或删除**：
  - 后置任务的依赖关系自动解除；
  - 通知后置任务负责人。

## 术语表

| 术语 | 说明 |
|------|------|
| 年度任务计划 | 客户×年度的组合容器，每个客户每年有一个独立的任务计划容器 |
| 摘要任务 | 有子任务的父任务，进度由子任务自动汇总 |
| 叶子任务 | 无子任务的末端任务，进度手动更新 |
| 前置任务 | 依赖关系中必须先完成/开始的任务 |
| 后置任务 | 依赖关系中依赖其他任务的任务 |
| FS | Finish-to-Start，完成-开始依赖 |
| SS | Start-to-Start，开始-开始依赖 |
| 活跃年度 | 当前正在使用的年度周期 |
| RBAC | Role-Based Access Control，基于角色的访问控制 |
| 细粒度权限 | 权限 = 资源 × 操作 × 数据范围 |
