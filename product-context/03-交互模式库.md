# 交互模式库

> 从实际需求中提炼的可复用交互模式，每个模式描述一个具体的用户操作流程。

## 字段说明

| 字段 | 说明 |
|------|------|
| **编号** | IP-001 格式，递增 |
| **名称** | 简洁的模式名称 |
| **触发条件** | 什么情况下使用此模式 |
| **流程描述** | 步骤列表：用户操作 → 系统响应 |
| **状态说明** | 各状态的视觉表现 |
| **异常处理** | 错误/边界情况的处理方式 |
| **适用场景** | 适用的业务上下文 |
| **引用案例** | 引用的需求路径（必填） |

## IP-001（示例）：批量操作前置分类校验

- **触发条件**：用户勾选多条记录后点击批量操作按钮，但所选项中存在因业务约束而不可执行的记录时。
- **流程描述**：
  1. 用户操作：勾选若干条记录，点击「批量操作」。
  2. 系统响应：对选中项进行分类统计（可执行数量 / 不可执行数量）。
  3. 系统响应：根据统计结果分三条分支展示对应弹窗：
     - **全部可执行**：标准确认弹窗；
     - **部分可执行**：弹窗内以警告文字说明跳过数量；
     - **全部不可执行**：阻断式弹窗，说明原因，仅提供「知道了」按钮。
  4. 用户操作（可执行场景）：点击确认执行。
  5. 系统响应：仅对可执行项执行操作，完成后展示结果提示。
- **状态说明**：
  - 全部可执行：标准确认弹窗，无额外提示；
  - 部分可执行：弹窗副文案使用警告色，明确告知跳过数量和原因；
  - 全部不可执行：阻断式弹窗，无操作按钮，用户只能关闭。
- **异常处理**：不可执行的判定依据由业务规则决定，不在此模式范围内。
- **适用场景**：批量操作中存在部分不可执行项，且不可执行原因需明确告知用户的场景。
- **引用案例**：`{你的需求路径}/`（待填入真实案例后替换）
