# 设计原则库

> 从实际需求案例中提炼的可复用设计原则，每条原则必须有实际案例支撑。

## 字段说明

| 字段 | 说明 |
|------|------|
| **编号** | DP-001 格式，递增 |
| **名称** | 简洁的原则名称（4-10 字） |
| **描述** | 1-3 句话说明原则内容 |
| **来源案例** | 引用的需求路径（必填） |
| **适用范围** | 适用的场景或组件类型 |
| **反例** | 违反此原则的典型做法 |

## DP-001（示例）：操作前置条件判断

- **描述**：为了避免用户进入操作流程后才发现无法继续，应当在用户触发操作时先校验前置条件，根据不同结果引导至不同的操作分支，而非统一进入同一个流程。
- **来源案例**：`{你的需求路径}/`（待填入真实案例后替换）
- **适用范围**：所有涉及操作前置条件判断的场景，特别是创建/编辑等复杂操作前。
- **反例**：用户点击操作按钮直接进入流程，填写大量信息后才被告知"该条件不满足，无法执行"。
